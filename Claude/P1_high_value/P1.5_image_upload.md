# P1.5 — Image upload to Supabase Storage

## Priority: P1

## Problem

The image block in `WikiPageEditor.tsx` only accepts a URL. Most users do not
have a CDN URL handy — they want to paste from clipboard or pick a file.
Supabase Storage (bucket `wiki_space_files`) is already in the stack.

## Acceptance criteria

- [ ] The image block editor shows a file input alongside the URL field
- [ ] Selecting a file uploads it to Supabase Storage and populates the `url` field
- [ ] During upload a spinner is shown and the input is disabled
- [ ] On upload error, a Norwegian error message is shown inside the block card
- [ ] Uploaded image URL is a public URL from the Supabase Storage bucket
- [ ] File size is validated client-side: reject files > 5 MB with a clear message
- [ ] Accepted MIME types: `image/jpeg`, `image/png`, `image/webp`, `image/gif`
- [ ] `npm run build` exits 0, zero TS errors

## Implementation

### Upload helper (add inside WikiPageEditor.tsx, not exported)

```tsx
async function uploadImageToStorage(
  file: File,
  orgId: string,
): Promise<string> {
  const MAX_BYTES = 5 * 1024 * 1024
  const ALLOWED = ['image/jpeg', 'image/png', 'image/webp', 'image/gif']

  if (file.size > MAX_BYTES) {
    throw new Error('Filen er for stor. Maksimal størrelse er 5 MB.')
  }
  if (!ALLOWED.includes(file.type)) {
    throw new Error('Filtypen støttes ikke. Bruk JPEG, PNG, WebP eller GIF.')
  }

  const ext = file.name.split('.').pop() ?? 'jpg'
  const path = `${orgId}/wiki-images/${crypto.randomUUID()}.${ext}`

  const { error } = await supabase.storage
    .from('wiki_space_files')
    .upload(path, file, { contentType: file.type, upsert: false })

  if (error) throw new Error(`Opplasting feilet: ${error.message}`)

  const { data } = supabase.storage
    .from('wiki_space_files')
    .getPublicUrl(path)

  return data.publicUrl
}
```

### Image block editor card changes

Locate the image block case in the editor. Add upload state and a file input:

```tsx
// State per block — use a Map keyed by block index to avoid re-renders
const [uploadingIndex, setUploadingIndex] = useState<number | null>(null)
const [uploadError, setUploadError] = useState<Record<number, string>>({})

// Inside image block card:
async function handleFileChange(
  e: React.ChangeEvent<HTMLInputElement>,
  blockIndex: number,
) {
  const file = e.target.files?.[0]
  if (!file) return
  setUploadingIndex(blockIndex)
  setUploadError((prev) => ({ ...prev, [blockIndex]: '' }))
  try {
    const url = await uploadImageToStorage(file, orgId)  // orgId from component props/context
    updateBlock(blockIndex, { ...block, url })
    markDirty()
  } catch (err) {
    setUploadError((prev) => ({
      ...prev,
      [blockIndex]: err instanceof Error ? err.message : 'Opplasting feilet',
    }))
  } finally {
    setUploadingIndex(null)
  }
}
```

In the image block JSX, add the file input **above** the URL input:

```tsx
<label className="flex cursor-pointer items-center gap-2 rounded-lg border border-dashed border-neutral-300 px-3 py-2 text-sm text-neutral-500 hover:border-neutral-400">
  {uploadingIndex === index ? (
    <span className="text-xs">Laster opp…</span>
  ) : (
    <>
      <Upload className="size-4" />
      <span className="text-xs">Last opp bilde (maks 5 MB)</span>
    </>
  )}
  <input
    type="file"
    accept="image/jpeg,image/png,image/webp,image/gif"
    className="sr-only"
    disabled={uploadingIndex === index}
    onChange={(e) => handleFileChange(e, index)}
  />
</label>
{uploadError[index] && (
  <p className="text-xs text-red-600">{uploadError[index]}</p>
)}
```

Keep the existing URL text input below the file input as a fallback for users
who do have a CDN URL.
