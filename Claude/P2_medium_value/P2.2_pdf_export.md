# P2.2 — PDF export

## Priority: P2

## Problem

There is no way to export a document to PDF. This is required for: printing HMS
procedures for physical binders, sending to Arbeidstilsynet, and sharing with
employees who lack platform access.

## Acceptance criteria

- [ ] A "Last ned PDF" button appears in the page view toolbar for published pages
- [ ] Clicking it opens a browser print dialog pre-styled for A4 output
- [ ] The printed/exported page includes: page title, space name, version number,
  publication date, and all block content
- [ ] Legal reference blocks render as formatted citations
- [ ] The platform navigation chrome (sidebar, header) is hidden in print
- [ ] Page footer shows: "Eksportert fra Klarert · v{version} · {date}"
- [ ] `npm run build` exits 0, zero TS errors

## Approach: CSS print stylesheet (no new dependency)

Do **not** add `@react-pdf/renderer` or Puppeteer for this feature. Use a CSS
print stylesheet and `window.print()`. This approach:
- Requires zero new dependencies
- Works offline
- Produces clean output from the existing `WikiBlockRenderer` markup

## Print stylesheet

```css
/* src/styles/print.css */
@media print {
  /* Hide all navigation chrome */
  nav,
  aside,
  header,
  [data-print-hide],
  .no-print {
    display: none !important;
  }

  /* Show print-only elements */
  [data-print-only] {
    display: block !important;
  }

  body {
    font-family: Georgia, 'Times New Roman', serif;
    font-size: 11pt;
    color: #000;
    background: #fff;
  }

  /* Page breaks */
  h1, h2, h3 { page-break-after: avoid; }
  table, figure { page-break-inside: avoid; }

  /* Links: show href */
  a[href]::after {
    content: ' (' attr(href) ')';
    font-size: 9pt;
    color: #555;
  }
  /* … but not internal links */
  a[href^="/"]::after { content: ''; }

  /* Law ref styling */
  .law-ref-block {
    border-left: 3pt solid #000;
    padding-left: 8pt;
    margin: 6pt 0;
  }
}
```

## Print button and metadata

In `WikiPageView.tsx`, add the print button to the toolbar:

```tsx
<button
  className="no-print inline-flex items-center gap-1.5 rounded-lg border border-neutral-200 px-3 py-1.5 text-sm hover:bg-neutral-50"
  onClick={() => window.print()}
  data-print-hide
>
  <Printer className="size-4" />
  Last ned PDF
</button>
```

Add a hidden print header above the page content:

```tsx
<div data-print-only className="hidden mb-6 border-b border-black pb-4">
  <h1 className="text-xl font-bold">{page.title}</h1>
  <p className="text-sm text-neutral-600">
    {space?.title} · Versjon {page.version} ·{' '}
    Publisert {page.updatedAt
      ? new Date(page.updatedAt).toLocaleDateString('no-NO')
      : '—'}
  </p>
</div>
```

Add a hidden print footer:

```tsx
<div data-print-only className="hidden mt-8 border-t border-black pt-4 text-xs text-neutral-500">
  Eksportert fra Klarert · v{page.version} ·{' '}
  {new Date().toLocaleDateString('no-NO')}
</div>
```

Make the `[data-print-only]` elements visible only via the print CSS above.

## Anti-patterns — do NOT do these

- Do **not** add `@react-pdf/renderer`, `jspdf`, or any PDF library — use
  CSS print + `window.print()`
- Do **not** add Puppeteer or server-side rendering for PDF
- Do **not** hide the print button from users with `documents.view` permission
