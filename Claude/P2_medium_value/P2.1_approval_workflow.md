# P2.1 — Approval workflow before publish

## Priority: P2

## Problem

The current lifecycle is `draft → published` with no gate. High-risk documents
(HMS handbook, privacy policy) should require a named reviewer to approve before
publication. This maps directly to IK-f §5 requirements for systematic review
and is a standard feature in Confluence and Guru.

## Acceptance criteria

- [ ] A page can optionally have `reviewRequired: true` and a `reviewerId` (user id)
- [ ] When `reviewRequired` is true, the "Lagre og publiser" button becomes "Send til godkjenning"
- [ ] Sending for review creates a `wiki_review_requests` DB record and an in-app notification
- [ ] The reviewer sees pending review requests in a new `/documents/reviews` page
- [ ] The reviewer can approve (publishes the page) or request changes (sends a comment back)
- [ ] Audit ledger records `submitted_for_review`, `approved`, `changes_requested` actions
- [ ] If `reviewRequired` is false, existing publish flow is unchanged
- [ ] `npm run build` exits 0, zero TS errors

## Database migration

```sql
CREATE TABLE IF NOT EXISTS wiki_review_requests (
  id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  org_id uuid NOT NULL REFERENCES organisations(id) ON DELETE CASCADE,
  page_id uuid NOT NULL REFERENCES wiki_pages(id) ON DELETE CASCADE,
  page_version int NOT NULL,
  requester_id uuid NOT NULL REFERENCES auth.users(id),
  reviewer_id uuid NOT NULL REFERENCES auth.users(id),
  status text NOT NULL DEFAULT 'pending'
    CHECK (status IN ('pending', 'approved', 'changes_requested')),
  reviewer_comment text,
  created_at timestamptz NOT NULL DEFAULT now(),
  resolved_at timestamptz
);

ALTER TABLE wiki_review_requests ENABLE ROW LEVEL SECURITY;

-- Requester can see their own requests
CREATE POLICY "requester_read" ON wiki_review_requests
  FOR SELECT USING (requester_id = auth.uid());

-- Reviewer can see requests assigned to them
CREATE POLICY "reviewer_read" ON wiki_review_requests
  FOR SELECT USING (reviewer_id = auth.uid());

-- Org admins can see all
CREATE POLICY "admin_read" ON wiki_review_requests
  FOR SELECT USING (
    EXISTS (
      SELECT 1 FROM org_members
      WHERE org_members.user_id = auth.uid()
        AND org_members.org_id = wiki_review_requests.org_id
        AND org_members.role = 'admin'
    )
  );

-- Only the requester can insert
CREATE POLICY "requester_insert" ON wiki_review_requests
  FOR INSERT WITH CHECK (requester_id = auth.uid());

-- Only the reviewer can update status
CREATE POLICY "reviewer_update" ON wiki_review_requests
  FOR UPDATE USING (reviewer_id = auth.uid());
```

Verify that `organisations` and `org_members` match the actual table names in
the codebase before running this migration.

## Publish button logic

```tsx

  </Button>

  </Button>
)}
```
