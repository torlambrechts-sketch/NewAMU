# P2.1 — Approval workflow before publish

## Priority: P2

## Problem

The current lifecycle is `draft → published` with no gate. High-risk documents
should require a named reviewer to approve before publication.

## Acceptance criteria

- [ ] Page can have `reviewRequired: true` and a `reviewerId`
- [ ] When set, "Lagre og publiser" becomes "Send til godkjenning"
- [ ] Submitting creates a `wiki_review_requests` record
- [ ] Reviewer sees pending requests at `/documents/reviews`
- [ ] Approving publishes the page; requesting changes sends comment back
- [ ] Audit ledger records `submitted_for_review`, `approved`, `changes_requested`
- [ ] Pages without `reviewRequired` publish directly unchanged
- [ ] `npm run build` exits 0

## Database migration

```sql
CREATE TABLE IF NOT EXISTS wiki_review_requests (
  id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  org_id uuid NOT NULL REFERENCES organisations(id) ON DELETE CASCADE,
  page_id uuid NOT NULL REFERENCES wiki_pages(id) ON DELETE CASCADE,
  page_version int NOT NULL,
  requester_id uuid NOT NULL REFERENCES auth.users(id),
  reviewer_id uuid NOT NULL REFERENCES auth.users(id),
  status text NOT NULL DEFAULT 'pending'
    CHECK (status IN ('pending', 'approved', 'changes_requested')),
  reviewer_comment text,
  created_at timestamptz NOT NULL DEFAULT now(),
  resolved_at timestamptz
);

ALTER TABLE wiki_review_requests ENABLE ROW LEVEL SECURITY;

CREATE POLICY "requester_read" ON wiki_review_requests FOR SELECT USING (requester_id = auth.uid());
CREATE POLICY "reviewer_read" ON wiki_review_requests FOR SELECT USING (reviewer_id = auth.uid());
CREATE POLICY "requester_insert" ON wiki_review_requests FOR INSERT WITH CHECK (requester_id = auth.uid());
CREATE POLICY "reviewer_update" ON wiki_review_requests FOR UPDATE USING (reviewer_id = auth.uid());
```


## Publish button

