# P2.4 — Save / publish feedback improvements

## Priority: P2

## Problem

Three related UX gaps in `WikiPageEditor.tsx`:
1. Buttons stay enabled during the network call — users double-click
2. The "Lagret" success banner auto-dismisses — users miss it
3. On save error the `WarningBox` is at the top but users are scrolled down

These are documented in `docs/DOCUMENTS_MODULE_UX_GUIDELINES.md` sections 1.2,
1.1, and 1.3.

## Acceptance criteria

- [ ] Save and publish buttons are disabled and show a spinner while the
  network call is in-flight
- [ ] "Lagret" banner stays visible until the user makes the next edit
  (dismissed by `markDirty`, not by a timer)
- [ ] On save error, the page scrolls to the error box automatically
- [ ] On publish error, same scroll-to-error behaviour
- [ ] "Prøv igjen" button inside the error box re-invokes the failed operation
- [ ] No TypeScript errors

## Implementation

### Fix 1 — In-progress state

Ensure a `saving` boolean state exists:
```tsx
const [saving, setSaving] = useState(false)
```

In the save handler (and publish handler), wrap the async call:
```tsx
async function handleSave() {
  setSaving(true)
  setSaveError(null)
  try {
    await updatePage(/* … */)
    setSavedMsg(true)   // or whatever the success state is called — check the file
  } catch (err) {
    setSaveError(err instanceof Error ? err.message : 'Lagring feilet')
    errorBoxRef.current?.scrollIntoView({ behavior: 'smooth', block: 'center' })
  } finally {
    setSaving(false)
  }
}
```

Disable both buttons while saving:
```tsx
<Button onClick={handleSave} disabled={saving}>
  {saving ? <Loader2 className="size-4 animate-spin" /> : null}
  Lagre
</Button>
```

Verify the `Button` component API by reading
`src/components/ui/Button.tsx` — it may already accept a `loading` prop.

### Fix 2 — Dismiss banner on next edit

Locate `markDirty`. Add a call to clear the saved state:
```tsx
function markDirty() {
  setDirty(true)
  setSavedMsg(false)   // clear banner on next edit
}
```

Remove any `setTimeout` that auto-dismisses the banner.

### Fix 3 — Scroll to error

Add a ref for the error box:
```tsx
const errorBoxRef = useRef<HTMLDivElement>(null)
```

Attach it to the `WarningBox` wrapper:
```tsx
{saveError && (
  <div ref={errorBoxRef}>
    <WarningBox>
      {saveError}
      <button
        className="mt-2 block text-xs underline"
        onClick={handleSave}
      >
        Prøv igjen
      </button>
    </WarningBox>
  </div>
)}
```

Scroll to the error box after setting the error state (as shown in Fix 1).

## Validation checklist

- [ ] `npm run build` exits 0, zero TS errors

- [ ] A spinner icon appears on the save button during the network call

- [ ] No auto-dismiss timer fires for the saved banner
- [ ] On simulated save failure, page scrolls to the error box
