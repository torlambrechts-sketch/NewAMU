# P2.4 — Save / publish feedback improvements

## Priority: P2

## Problem

Three UX gaps in `WikiPageEditor.tsx`:
1. Buttons stay enabled during network call — users double-click
2. "Lagret" banner auto-dismisses — users miss it
3. On save error the `WarningBox` is at top but user is scrolled down

## Acceptance criteria

- [ ] Save and publish buttons disabled + spinner while call is in-flight
- [ ] "Lagret" banner stays until next edit (`markDirty` clears it, no timer)
- [ ] On save error, page scrolls to error box automatically
- [ ] "Prøv igjen" button inside error box retries the operation
- [ ] No TypeScript errors

## Files to modify

| File | Change |
|---|---|
| `src/pages/documents/WikiPageEditor.tsx` | All three fixes |

## Implementation

### Fix 1 — saving state

```tsx
const [saving, setSaving] = useState(false)

async function handleSave() {
  setSaving(true); setSaveError(null)
  try {
    await updatePage(/* … */)
    setSavedMsg(true)
  } catch (err) {
    setSaveError(err instanceof Error ? err.message : 'Lagring feilet')
    errorBoxRef.current?.scrollIntoView({ behavior: 'smooth', block: 'center' })
  } finally { setSaving(false) }
}
```

Disable buttons: `<Button onClick={handleSave} disabled={saving}>`

### Fix 2 — clear banner on next edit

```tsx
function markDirty() {
  setDirty(true)
  setSavedMsg(false)   // clear banner
}
```

Remove any `setTimeout` auto-dismiss.

### Fix 3 — scroll to error + retry

```tsx
const errorBoxRef = useRef<HTMLDivElement>(null)

{saveError && (
  <div ref={errorBoxRef}>
    <WarningBox>
      {saveError}
      <button className="mt-2 block text-xs underline" onClick={handleSave}>
        Prøv igjen
      </button>
    </WarningBox>
  </div>
)}
```
