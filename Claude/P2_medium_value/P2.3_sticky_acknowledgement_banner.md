# P2.3 — Sticky "must acknowledge" banner on page view

## Priority: P2

## Problem

When a document requires acknowledgement, the `acknowledgement_footer` module
block is at the very bottom of the page. Users who skim the content without
scrolling all the way down never see it and never sign. There is no indicator
that a signature is required until the user reaches the footer.

## Acceptance criteria

- [ ] When a published page has `requiresAcknowledgement: true` AND the current
  user has not yet signed the current version, a sticky bottom banner appears
- [ ] Banner text: "Dette dokumentet krever din bekreftelse ↓ Scroll ned for å signere"
- [ ] Clicking the banner scrolls the page to the `acknowledgement_footer` block
- [ ] The banner disappears once the user has scrolled the footer into view
  OR has already acknowledged the current version
- [ ] Users who have already signed see no banner
- [ ] The banner does not appear on draft or archived pages
- [ ] The banner does not appear for users outside the `acknowledgementAudience`
- [ ] `npm run build` exits 0, zero TS errors

## Implementation

### Detect if the current user needs to acknowledge

From existing code in `WikiPageView`, there should already be logic to check
whether the current user has a receipt for the current page version. If not:

```tsx
const userReceipt = docs.receipts.find(
  (r) => r.pageId === page.id && r.pageVersion === page.version && r.userId === currentUserId
)
const needsAck =
  page.requiresAcknowledgement &&
  page.status === 'published' &&
  !userReceipt
```

Verify the actual field names by reading `ComplianceReceipt` in `documents.ts`.

### Ref for the acknowledgement footer block

```tsx
const ackFooterRef = useRef<HTMLDivElement>(null)
```

Attach this ref to the rendered `acknowledgement_footer` module block in
`WikiBlockRenderer.tsx` or by wrapping it in `WikiPageView.tsx`.

### Scroll-into-view detection

```tsx
const [ackVisible, setAckVisible] = useState(false)

useEffect(() => {
  if (!needsAck || !ackFooterRef.current) return
  const observer = new IntersectionObserver(
    ([entry]) => setAckVisible(entry?.isIntersecting ?? false),
    { threshold: 0.5 }
  )
  observer.observe(ackFooterRef.current)
  return () => observer.disconnect()
}, [needsAck])
```

### Sticky banner JSX

Render this at the bottom of the page JSX (as a sibling of the main content,
not inside it):

```tsx
{needsAck && !ackVisible && (
  <div
    role="status"
    className="fixed bottom-0 left-0 right-0 z-50 flex items-center justify-center gap-3
               border-t border-amber-300 bg-amber-50 px-4 py-3 shadow-lg"
  >
    <span className="text-sm font-medium text-amber-900">
      Dette dokumentet krever din bekreftelse
    </span>
    <button
      className="rounded-lg bg-amber-600 px-4 py-1.5 text-sm font-medium text-white hover:bg-amber-700"
      onClick={() => ackFooterRef.current?.scrollIntoView({ behavior: 'smooth' })}
    >
      Scroll ned ↓
    </button>
  </div>
)}
```

### Audience check

Before showing the banner, verify the current user is in the target audience:

```tsx
function isInAudience(page: WikiPage, userId: string, members: Member[]): boolean {
  const audience = page.acknowledgementAudience ?? 'all_employees'
  if (audience === 'all_employees') return true
  const member = members.find((m) => m.userId === userId)
  if (!member) return false
  if (audience === 'leaders_only') return member.role === 'admin' || member.role === 'leader'
  if (audience === 'safety_reps_only') return member.role === 'safety_rep'
  if (audience === 'department') return member.departmentId === page.acknowledgementDepartmentId
  return true
}
```

Verify `Member` field names against the actual type in the codebase.
