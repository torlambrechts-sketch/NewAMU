# P0.2 — Remove or disable non-functional field tiles in DocumentEditorWorkbench

## Priority: P0 — fix before shipping any new feature

## Problem

`src/components/documents/DocumentEditorWorkbench.tsx` renders six "Utfyllbare
felt" tiles (text field, signature, initials, date, checkbox, dropdown). Each
tile has an orange interactive border and `cursor-pointer` styling — but **none
has an `onClick` handler**. Clicking them does nothing. This destroys user
confidence in the workbench.

The correct short-term fix is to **visually disable** the tiles with a "coming
soon" tooltip until the insertion logic is built. Do **not** silently keep them
clickable with zero effect.

## Acceptance criteria

- [ ] Tiles are visible but clearly non-interactive (opacity reduced, cursor default)
- [ ] Hovering a tile shows a tooltip: "Kommer snart"
- [ ] No `onClick` handler is wired to any tile
- [ ] The orange interactive border is removed or replaced with a neutral border
- [ ] No TypeScript errors introduced
- [ ] The rest of the workbench (specification form, version history) is untouched

## Implementation steps

### Step 1 — Read the file

Read `src/components/documents/DocumentEditorWorkbench.tsx` in full. Locate:
- The six tile definitions (array or inline JSX)
- Any `onClick` props on tile elements
- The existing CSS classes that create the orange border / pointer cursor

### Step 2 — Replace tile styling

For every tile element, replace the existing interactive classes with:

```tsx
// Before (example pattern — verify against actual code)
<div className="cursor-pointer rounded-lg border-2 border-orange-400 p-3 ...">

// After
<div
  className="cursor-not-allowed rounded-lg border border-neutral-200 p-3 opacity-50 ..."
  title="Kommer snart"
  aria-disabled="true"
>
```

Key class changes:
- `cursor-pointer` → `cursor-not-allowed`
- `border-orange-400` (or similar) → `border-neutral-200`
- Add `opacity-50`
- Add `title="Kommer snart"` on the element for a native browser tooltip
- Add `aria-disabled="true"` for accessibility

### Step 3 — Remove any onClick handler

If any tile has an `onClick` prop (even a no-op), remove it entirely.
An `aria-disabled` element must not respond to click events.

If a click event needs to be suppressed on a parent:
```tsx
onClick={(e) => e.stopPropagation()}
```
Do not add this unless required by an existing parent handler.

### Step 4 — Update section label (optional but recommended)

If the section is labelled "Utfyllbare felt", add a sub-label:
```tsx
<p className="text-xs text-neutral-400">Tilgjengelig i en fremtidig versjon</p>
```

## Anti-patterns — do NOT do these

- Do **not** delete the tiles entirely — they signal planned functionality
- Do **not** add an `onClick` that opens a toast saying "ikke tilgjengelig" —
  a disabled element should not be clickable at all
- Do **not** change any other part of the workbench component
