# P0.3 — Wire unsaved-changes navigation guard

## Priority: P0 — fix before shipping any new feature

## Problem

`src/pages/documents/WikiPageEditor.tsx` tracks a `dirty` boolean but **no
navigation guard is wired**: clicking back or any sidebar link while
`dirty === true` silently discards all unsaved changes.

## Acceptance criteria

- [ ] Navigating away while `dirty === true` shows a Norwegian confirmation dialog
- [ ] Confirming allows navigation; cancelling keeps user on the page
- [ ] Browser `beforeunload` also triggers a warning (tab close / refresh)
- [ ] `dirty === false` → navigation is unblocked
- [ ] After successful save, `dirty` resets and navigation is unblocked
- [ ] No TypeScript errors

## Implementation

### React Router blocker

```tsx
import { useBlocker } from 'react-router-dom'

const blocker = useBlocker(
  ({ currentLocation, nextLocation }) =>
    dirty && currentLocation.pathname !== nextLocation.pathname
)
```

### Confirmation dialog

```tsx
{blocker.state === 'blocked' && (
  <div role="dialog" aria-modal="true"
    className="fixed inset-0 z-[80] flex items-center justify-center bg-black/50">
    <div className="w-full max-w-sm rounded-2xl bg-white p-6 shadow-2xl">
      <h2 className="text-base font-semibold text-neutral-900">Ulagrede endringer</h2>
      <p className="mt-2 text-sm text-neutral-600">
        Du har endringer som ikke er lagret. Hvis du forlater siden nå, vil
        endringene gå tapt.
      </p>
      <div className="mt-6 flex justify-end gap-3">
        <button onClick={() => blocker.reset()}
          className="rounded-lg border border-neutral-200 px-4 py-2 text-sm font-medium">
          Bli på siden
        </button>
        <button onClick={() => blocker.proceed()}
          className="rounded-lg bg-red-600 px-4 py-2 text-sm font-medium text-white hover:bg-red-700">
          Forlat siden
        </button>
      </div>
    </div>
  </div>
)}
```

### beforeunload listener

```tsx
useEffect(() => {
  if (!dirty) return
  const handler = (e: BeforeUnloadEvent) => { e.preventDefault() }
  window.addEventListener('beforeunload', handler)
  return () => window.removeEventListener('beforeunload', handler)
}, [dirty])
```

## Anti-patterns

- Do **not** use `window.confirm()` — use the inline dialog
- Do **not** set `e.returnValue` to a custom string — browsers ignore it
- Do **not** block navigation when `dirty === false`
