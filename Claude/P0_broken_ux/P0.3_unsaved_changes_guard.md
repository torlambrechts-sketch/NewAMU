# P0.3 ŌĆö Wire unsaved-changes navigation guard

## Priority: P0 ŌĆö fix before shipping any new feature

## Problem

`src/pages/documents/WikiPageEditor.tsx` tracks a `dirty` boolean ŌĆö it is set
`true` when the user edits any block or setting. But **no navigation guard is
wired**: clicking the browser back button or any sidebar link while `dirty ===
true` silently discards all unsaved changes.

The hook `useDirtyGuard` exists in the codebase. It just needs to be called.

## Acceptance criteria

- [ ] Navigating away while `dirty === true` shows a confirmation dialog
- [ ] The dialog text is in Norwegian Bokm├źl
- [ ] Confirming the dialog allows navigation; cancelling keeps the user on the page
- [ ] The browser `beforeunload` event also triggers a warning (tab close / refresh)
- [ ] When `dirty === false` (no changes) navigation is unblocked
- [ ] After a successful save, `dirty` is reset and navigation is unblocked
- [ ] No TypeScript errors

## Implementation steps

### Step 1 ŌĆö Locate useDirtyGuard

Search for `useDirtyGuard` in the codebase. Read the file where it is defined.
Understand its API: does it accept a boolean, a callback, or both?

### Step 2A ŌĆö If useDirtyGuard already handles both router and browser guards

Add the single call inside `WikiPageEditor`:

```tsx
useDirtyGuard(dirty)
```

Import it from wherever it is defined.

### Step 2B ŌĆö If useDirtyGuard only handles browser beforeunload

Add the router blocker separately using React Router 7's `useBlocker`:

```tsx
import { useBlocker } from 'react-router-dom'

const blocker = useBlocker(
  ({ currentLocation, nextLocation }) =>
    dirty && currentLocation.pathname !== nextLocation.pathname
)
```

Then render the confirmation dialog when `blocker.state === 'blocked'`:

```tsx
{blocker.state === 'blocked' && (
  <div
    role="dialog"
    aria-modal="true"
    aria-labelledby="dirty-guard-title"
    className="fixed inset-0 z-[80] flex items-center justify-center bg-black/50"
  >
    <div className="w-full max-w-sm rounded-2xl bg-white p-6 shadow-2xl">
      <h2 id="dirty-guard-title" className="text-base font-semibold text-neutral-900">
        Ulagrede endringer
      </h2>
      <p className="mt-2 text-sm text-neutral-600">
        Du har endringer som ikke er lagret. Hvis du forlater siden n├ź, vil
        endringene g├ź tapt.
      </p>
      <div className="mt-6 flex justify-end gap-3">
        <button
          className="rounded-lg border border-neutral-200 px-4 py-2 text-sm font-medium text-neutral-700 hover:bg-neutral-50"
          onClick={() => blocker.reset()}
        >
          Bli p├ź siden
        </button>
        <button
          className="rounded-lg bg-red-600 px-4 py-2 text-sm font-medium text-white hover:bg-red-700"
          onClick={() => blocker.proceed()}
        >
          Forlat siden
        </button>
      </div>
    </div>
  </div>
)}
```

### Step 3 ŌĆö Add beforeunload listener

Whether or not `useDirtyGuard` already covers this, verify a `beforeunload`
listener exists. If not, add it:

```tsx
useEffect(() => {
  if (!dirty) return
  const handler = (e: BeforeUnloadEvent) => {
    e.preventDefault()
  }
  window.addEventListener('beforeunload', handler)
  return () => window.removeEventListener('beforeunload', handler)
}, [dirty])
```

Note: modern browsers show their own generic text for `beforeunload` ŌĆö do not
try to set `e.returnValue` to a custom string; it is ignored.

### Step 4 ŌĆö Verify dirty reset on save

Confirm that `dirty` is set to `false` after a successful save. The existing
save handler should call `setDirty(false)` or equivalent. If it does not, add
it in the `finally` block of the save function.

## Anti-patterns ŌĆö do NOT do these

- Do **not** use `window.confirm()` directly ŌĆö use the inline dialog above
- Do **not** set `e.returnValue` to a Norwegian string in `beforeunload` ŌĆö
  browsers ignore custom strings and this pattern is deprecated
- Do **not** block navigation when `dirty === false`
- Do **not** modify the `useBlocker` call condition to also block same-path
  navigation (tab switching within the editor)
