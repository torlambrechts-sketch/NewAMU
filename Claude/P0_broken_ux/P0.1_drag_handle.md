# P0.1 — Implement drag-and-drop block reorder

## Priority: P0 — fix before shipping any new feature

## Problem

The `GripVertical` icon from Lucide React is rendered on every content block
inside `src/pages/documents/WikiPageEditor.tsx`. It has interactive styling
(cursor-grab) but **no drag-and-drop implementation**. Users drag it repeatedly,
nothing happens, and trust in the editor collapses. The up/down arrow buttons
work but are easy to miss.

A non-functional affordance is worse than no affordance.

## Acceptance criteria

- [ ] Blocks reorder via the grip handle using pointer drag
- [ ] A translucent drag overlay shows the block being dragged
- [ ] Drop target is visually highlighted while dragging
- [ ] `page.blocks` array updates immediately on drop
- [ ] `markDirty()` is called after every reorder
- [ ] Up/down arrow buttons continue to work (do not remove them)
- [ ] `npm run build` exits 0 with zero TS errors
- [ ] No `as any` or `@ts-ignore` introduced
- [ ] Works with both mouse and touch (PointerSensor covers both)

## Implementation steps

### Step 1 — Read the file

Read `src/pages/documents/WikiPageEditor.tsx` in full. Locate:
- The array map that renders block cards
- The `GripVertical` element
- The `markDirty` call site
- The up/down arrow handlers

### Step 2 — Add imports

```tsx
import {
  DndContext,
  closestCenter,
  PointerSensor,
  useSensor,
  useSensors,
  DragEndEvent,
  DragOverlay,
  DragStartEvent,
} from '@dnd-kit/core'
import {
  SortableContext,
  verticalListSortingStrategy,
  arrayMove,
  useSortable,
} from '@dnd-kit/sortable'
import { CSS } from '@dnd-kit/utilities'
```

### Step 3 — Add drag state to the editor component

```tsx
const [activeBlockIndex, setActiveBlockIndex] = useState<number | null>(null)
const sensors = useSensors(useSensor(PointerSensor))
```

### Step 4 — Add drag handlers

```tsx
function handleDragStart(event: DragStartEvent) {
  setActiveBlockIndex(Number(event.active.id))
}

function handleDragEnd(event: DragEndEvent) {
  setActiveBlockIndex(null)
  const { active, over } = event
  if (!over || active.id === over.id) return
  const oldIndex = Number(active.id)
  const newIndex = Number(over.id)
  setPage((prev) => ({ ...prev, blocks: arrayMove(prev.blocks, oldIndex, newIndex) }))
  markDirty()
}
```

### Step 5 — Create a SortableBlockCard wrapper

Add this component **above** the main editor component in the same file:

```tsx
function SortableBlockCard({
  index,
  children,
}: {
  index: number
  children: (dragHandleProps: React.HTMLAttributes<HTMLElement>) => React.ReactNode
}) {
  const { attributes, listeners, setNodeRef, transform, transition, isDragging } =
    useSortable({ id: String(index) })

  const style: React.CSSProperties = {
    transform: CSS.Transform.toString(transform),
    transition,
    opacity: isDragging ? 0.4 : 1,
    position: 'relative',
    zIndex: isDragging ? 10 : undefined,
  }

  return (
    <div ref={setNodeRef} style={style}>
      {children({ ...listeners, ...attributes })}
    </div>
  )
}
```

### Step 6 — Wrap the block list

In the JSX where `page.blocks.map(...)` is called:

```tsx
<DndContext
  sensors={sensors}
  collisionDetection={closestCenter}
  onDragStart={handleDragStart}
  onDragEnd={handleDragEnd}
>
  <SortableContext
    items={page.blocks.map((_, i) => String(i))}
    strategy={verticalListSortingStrategy}
  >
    {page.blocks.map((block, index) => (
      <SortableBlockCard key={index} index={index}>
        {(dragHandleProps) => (
          <BlockCard
            block={block}
            index={index}
            dragHandleProps={dragHandleProps}  {/* pass down to the grip icon */}
            {/* …rest of existing props */}
          />
        )}
      </SortableBlockCard>
    ))}
  </SortableContext>

  <DragOverlay>
    {activeBlockIndex !== null && page.blocks[activeBlockIndex] ? (
      <div className="rounded-xl border border-neutral-300 bg-white/90 p-4 shadow-xl">
        <p className="text-sm text-neutral-500">Blokk {activeBlockIndex + 1}</p>
      </div>
    ) : null}
  </DragOverlay>
</DndContext>
```

### Step 7 — Pass dragHandleProps to the GripVertical element

Inside the block card (wherever `GripVertical` is rendered):

```tsx
<GripVertical
  className="size-4 cursor-grab text-neutral-400 active:cursor-grabbing"
  {...dragHandleProps}
/>
```

Do not attach `dragHandleProps` to the entire card — only the grip icon.

## Anti-patterns — do NOT do these

- Do **not** use the HTML5 native `draggable` attribute — it conflicts with
  PointerSensor and breaks on touch screens
- Do **not** remove the up/down arrow buttons
- Do **not** mutate `page.blocks` directly; always use `setPage` with a
  new array reference
- Do **not** use block content as the sortable id — use the numeric index
  stringified
- Do **not** store `activeBlockIndex` as a block reference — store the
  integer index so it stays correct after any re-render
