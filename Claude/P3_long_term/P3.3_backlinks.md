# P3.3 — Cross-document backlinks

## Priority: P3

## Problem

No way to link between wiki pages or see which pages reference a given page.

## Acceptance criteria

- [ ] Typing `[[` in TipTap opens an autocomplete page picker
- [ ] Selecting inserts `<a href="/documents/page/:id">Title</a>`
- [ ] Backlinks stored in `wiki_page_links` on publish
- [ ] Page view shows "Referert av" section
- [ ] `ON DELETE CASCADE` cleans up link records
- [ ] `npm run build` exits 0

## Database migration

```sql
CREATE TABLE IF NOT EXISTS wiki_page_links (
  id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  org_id uuid NOT NULL REFERENCES organisations(id) ON DELETE CASCADE,
  source_page_id uuid NOT NULL REFERENCES wiki_pages(id) ON DELETE CASCADE,
  target_page_id uuid NOT NULL REFERENCES wiki_pages(id) ON DELETE CASCADE,
  UNIQUE (source_page_id, target_page_id)
);
ALTER TABLE wiki_page_links ENABLE ROW LEVEL SECURITY;
CREATE POLICY "org_member_read" ON wiki_page_links FOR SELECT USING (
  org_id IN (SELECT org_id FROM org_members WHERE user_id = auth.uid())
);
CREATE POLICY "system_upsert" ON wiki_page_links FOR ALL USING (
  org_id IN (SELECT org_id FROM org_members WHERE user_id = auth.uid())
);
```

## Backlink extraction on publish

In `publishPage` in `useDocuments.tsx`, after saving:
1. Extract all `href` values from `<a>` tags in `text` blocks
2. Filter for `/documents/page/` links, extract page IDs
3. Delete existing `wiki_page_links` where `source_page_id = page.id`
4. Insert new rows for each target page ID
