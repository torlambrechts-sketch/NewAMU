# P3.3 ŌĆö Cross-document backlinks and internal linking

## Priority: P3

## Problem

Procedures often reference each other ("se ogs├ź avviksrutinen"). There is no
way to link between wiki pages or see which pages reference a given page.
Notion and Roam-style backlinks provide a knowledge-graph view that makes the
document system more navigable.

## Acceptance criteria

- [ ] In TipTap editor, typing `[[` opens an autocomplete picker for wiki pages
- [ ] Selecting a page inserts a link block: `<a href="/documents/page/:id">Title</a>`
- [ ] Backlinks are stored in `wiki_page_links` when a page is published
- [ ] The page view shows a "Referert av" section listing pages that link to it
- [ ] Deleting a page cleans up outgoing link records (`ON DELETE CASCADE`)
- [ ] `npm run build` exits 0, zero TS errors

## Database migration

```sql
CREATE TABLE IF NOT EXISTS wiki_page_links (
  id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  org_id uuid NOT NULL REFERENCES organisations(id) ON DELETE CASCADE,
  source_page_id uuid NOT NULL REFERENCES wiki_pages(id) ON DELETE CASCADE,
  target_page_id uuid NOT NULL REFERENCES wiki_pages(id) ON DELETE CASCADE,
  UNIQUE (source_page_id, target_page_id)
);

ALTER TABLE wiki_page_links ENABLE ROW LEVEL SECURITY;

CREATE POLICY "org_member_read" ON wiki_page_links
  FOR SELECT USING (
    org_id IN (SELECT org_id FROM org_members WHERE user_id = auth.uid())
  );

CREATE POLICY "system_upsert" ON wiki_page_links
  FOR ALL USING (
    org_id IN (SELECT org_id FROM org_members WHERE user_id = auth.uid())
  );
```

## TipTap [[ autocomplete

Use TipTap's `Suggestion` extension (already in the TipTap package) to trigger
on `[[`:

```tsx
import { Extension } from '@tiptap/core'
import Suggestion from '@tiptap/suggestion'

// Read TipTapRichTextEditor.tsx first to understand how to pass pages list in
// and how extensions are composed. Do not invent the extension API.
```

The autocomplete items come from `docs.pages.filter(p => p.status === 'published')`.
On selection, insert a link node pointing to `/documents/page/:id`.

## Backlink extraction on publish

In `publishPage` (in `useDocuments.tsx`), after saving the page:

1. Extract all `href` values from `<a>` tags in `text` blocks
2. Filter for `/documents/page/` links and extract page IDs
3. Delete all existing `wiki_page_links` where `source_page_id = page.id`
4. Insert new rows for each extracted target page ID
