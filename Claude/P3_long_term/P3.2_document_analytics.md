# P3.2 — Document view analytics

## Priority: P3

## Problem

There is no way to know whether employees have read key HMS documents (beyond
acknowledgement receipts). View tracking provides a softer signal: who opened
the page, when, and how often. This is valuable for proving employee awareness
and for identifying documents that are never read.

## Acceptance criteria

- [ ] Every page view by an authenticated user is recorded in `wiki_page_views`
- [ ] Duplicate views within a 1-hour window for the same user+page are ignored
- [ ] Document owners / admins can see a "Visninger" tab on each page with:
  - Total unique viewers
  - Views in the last 30 days
  - A list of most recent views (name, date) capped at 50 rows
- [ ] The `ComplianceDashboard` shows "Sett av X av Y ansatte" alongside
  acknowledgement receipts
- [ ] View data is never shown to the viewer themselves in a way that creates
  a surveillance concern — aggregate only for non-admins
- [ ] `npm run build` exits 0, zero TS errors

## Database migration

```sql
CREATE TABLE IF NOT EXISTS wiki_page_views (
  id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  org_id uuid NOT NULL REFERENCES organisations(id) ON DELETE CASCADE,
  page_id uuid NOT NULL REFERENCES wiki_pages(id) ON DELETE CASCADE,
  user_id uuid NOT NULL REFERENCES auth.users(id) ON DELETE CASCADE,
  viewed_at timestamptz NOT NULL DEFAULT now()
);

ALTER TABLE wiki_page_views ENABLE ROW LEVEL SECURITY;

-- Users can insert their own views
CREATE POLICY "self_insert" ON wiki_page_views
  FOR INSERT WITH CHECK (user_id = auth.uid());

-- Admins can read all views in their org
CREATE POLICY "admin_read" ON wiki_page_views
  FOR SELECT USING (
    EXISTS (
      SELECT 1 FROM org_members
      WHERE org_members.user_id = auth.uid()
        AND org_members.org_id = wiki_page_views.org_id
        AND org_members.role = 'admin'
    )
  );

-- Index for fast per-page queries
CREATE INDEX IF NOT EXISTS wiki_page_views_page_idx
  ON wiki_page_views (page_id, viewed_at DESC);
```

## Record view on mount (WikiPageView.tsx)

```tsx
useEffect(() => {
  if (!page || page.status !== 'published' || !currentUserId) return

  // Dedup: don't re-record within 1 hour
  const key = `view_${page.id}`
  const last = sessionStorage.getItem(key)
  if (last && Date.now() - Number(last) < 3_600_000) return
  sessionStorage.setItem(key, String(Date.now()))

  supabase.from('wiki_page_views').insert({
    org_id: orgId,
    page_id: page.id,
    user_id: currentUserId,
  }).then(() => {})  // fire-and-forget
}, [page?.id, page?.status, currentUserId, orgId])
```
