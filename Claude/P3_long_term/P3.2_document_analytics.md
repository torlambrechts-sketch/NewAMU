# P3.2 — Document view analytics

## Priority: P3

## Problem

No way to know whether employees have read key HMS documents beyond ack
receipts. View tracking provides a softer awareness signal.

## Acceptance criteria

- [ ] Every page view by an authenticated user recorded in `wiki_page_views`
- [ ] Duplicate views within 1 hour for same user+page ignored (sessionStorage)
- [ ] Admins see a "Visninger" tab: unique viewers, views last 30 days, recent list
- [ ] `ComplianceDashboard` shows "Sett av X av Y ansatte"
- [ ] Aggregate only for non-admins (no surveillance concern)
- [ ] `npm run build` exits 0

## Files to create

| File | Purpose |

## Database migration

```sql
CREATE TABLE IF NOT EXISTS wiki_page_views (
  id uuid PRIMARY KEY DEFAULT gen_random_uuid(),
  org_id uuid NOT NULL REFERENCES organisations(id) ON DELETE CASCADE,
  page_id uuid NOT NULL REFERENCES wiki_pages(id) ON DELETE CASCADE,
  user_id uuid NOT NULL REFERENCES auth.users(id) ON DELETE CASCADE,
  viewed_at timestamptz NOT NULL DEFAULT now()
);
ALTER TABLE wiki_page_views ENABLE ROW LEVEL SECURITY;
CREATE POLICY "self_insert" ON wiki_page_views FOR INSERT WITH CHECK (user_id = auth.uid());
CREATE POLICY "admin_read" ON wiki_page_views FOR SELECT USING (
  EXISTS (SELECT 1 FROM org_members WHERE user_id = auth.uid()
    AND org_id = wiki_page_views.org_id AND role = 'admin')
);
CREATE INDEX IF NOT EXISTS wiki_page_views_page_idx ON wiki_page_views (page_id, viewed_at DESC);
```

## Record view on mount

```tsx
useEffect(() => {
  if (!page || page.status !== 'published' || !currentUserId) return
  const key = `view_${page.id}`
  const last = sessionStorage.getItem(key)
  if (last && Date.now() - Number(last) < 3_600_000) return
  sessionStorage.setItem(key, String(Date.now()))
  supabase.from('wiki_page_views').insert({ org_id: orgId, page_id: page.id, user_id: currentUserId }).then(() => {})
}, [page?.id, page?.status, currentUserId, orgId])
```
