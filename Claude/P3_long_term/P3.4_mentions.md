# P3.4 — @mentions in documents and comments

## Priority: P3

## Problem

Document reviewers and authors need to tag colleagues in comments or document
bodies. There is no @mention mechanism. This is required for approval workflows
(P2.1) and inline comments (P2.5) to be fully useful.

**Dependency:** Implement P2.5 (inline comments) before this spec.

## Acceptance criteria

- [ ] Typing `@` in TipTap or in a comment input opens a people picker
- [ ] Selecting a person inserts a styled `@Name` mention chip
- [ ] The mentioned user receives an in-app notification
- [ ] Mention chips are stored as `<span data-mention data-user-id="uuid">@Name</span>`
  in the sanitised HTML
- [ ] DOMPurify allows `data-mention` and `data-user-id` attributes
- [ ] `npm run build` exits 0, zero TS errors

## TipTap Mention extension

TipTap provides `@tiptap/extension-mention` — verify it is already in
`package.json`. If not, add it:

```bash
npm install @tiptap/extension-mention
```

Configure it with the org members list:

```tsx
import Mention from '@tiptap/extension-mention'
import { ReactRenderer } from '@tiptap/react'
// See TipTap docs for MentionList component pattern

Mention.configure({
  HTMLAttributes: {
    class: 'mention-chip rounded bg-[#1a3d32]/10 px-1 py-0.5 text-[#1a3d32] font-medium',
  },
  suggestion: {
    items: ({ query }: { query: string }) =>
      members
        .filter((m) => m.name.toLowerCase().includes(query.toLowerCase()))
        .slice(0, 8),
    render: () => { /* ReactRenderer pattern — follow TipTap docs exactly */ },
  },
})
```

## DOMPurify config update

Find where `DOMPurify.sanitize` is called. Add `data-mention` and `data-user-id`
to the allowed attributes:

```ts
DOMPurify.sanitize(html, {
  ADD_ATTR: ['data-mention', 'data-user-id'],
  // ... existing config
})
```

## Notification on mention

After saving a page or comment that contains mention chips, extract
`data-user-id` values and insert rows into the existing notifications table
(find the table name by searching for notification inserts in the codebase).
