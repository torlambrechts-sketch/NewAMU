# P1.4 — Ingen eksport av signert referat (PDF / utskrift)

## Prioritet: P1 (Høy daglig påvirkning)

## Problem

Etter at et AMU-møte er signert og låst, finnes det ingen måte å eksportere
referatet. AML §7-2(6) krever at referater «gjøres tilgjengelige» for alle
parter og for Arbeidstilsynet på forespørsel.

I dag er referatene kun tilgjengelige via Klarert-plattformen. Organisasjoner
trenger en PDF de kan arkivere i eget system, sende til fagforeninger og
verneombud, og fremlegge ved tilsyn.

Det finnes ingen `window.print()`-optimert visning, ingen PDF-generering og
ingen eksportknapp.

### Referansefiler (les disse FØR du redigerer noe)

- `modules/amu/AmuDetailView.tsx` — linjene 677–744 (signeringsfanen)
- `modules/amu/useAmu.ts` — linjene 84–106 (`getMeeting`), 366–384

  (`loadDecisionsForMeeting`)

- `package.json` — kontroller at ingen PDF-lib er tilgjengelig (ingen
  `jspdf`, ingen `pdfmake`, ingen `@react-pdf/renderer`)

## Acceptance criteria

- [ ] I signeringsfanen finnes en "Eksporter referat"-knapp synlig for alle
  brukere (ikke kun `amu.manage`) når møtet er signert.
- [ ] Klikk åpner en ny fane (`window.open`) med en print-optimert HTML-side
  som inneholder: møtetittel, dato, sted, deltakerliste med roller, saksliste
  med vedtak, referat-tekst, møteleder og signeringstidspunkt.
- [ ] HTML-siden starter automatisk `window.print()` etter 500 ms.
- [ ] Siden bruker CSS `@media print` for å skjule nettleserkrom.
- [ ] Ingen npm-pakker legges til.
- [ ] Siden genereres 100 % klientsiden — ingen serverrunder.
- [ ] `npm run build` exits 0.

## Implementeringstrinn

### Trinn 1 — Ny hjelpefunksjon `buildPrintHtml`

Legg til følgende funksjon i `AmuDetailView.tsx` FØR komponent-funksjonen
(ca. linje 77):

```tsx
function buildPrintHtml(opts: {
  title: string
  dateLabel: string
  location: string
  participants: AmuParticipant[]
  userLabel: (id: string) => string
  agenda: AmuAgendaItem[]
  decisionByAgenda: Record<string, AmuDecision | null>
  minutesDraft: string
  chairName: string | null
  chairSignedAt: string | null
}): string {
  const {
    title, dateLabel, location, participants,
    userLabel, agenda, decisionByAgenda,
    minutesDraft, chairName, chairSignedAt,
  } = opts

  const roleLabel: Record<AmuParticipant['role'], string> = {
    employer_rep: 'Arbeidsgivers representant',
    employee_rep: 'Arbeidstakerrepresentant',
    safety_deputy: 'Verneombud',
    bht: 'BHT',
    secretary: 'Referent',
  }

  const participantsHtml = participants
    .map(
      (p) =>
        `<tr><td>${userLabel(p.user_id)}</td><td>${roleLabel[p.role]}</td>` +
        `<td>${p.present ? 'Ja' : 'Nei'}</td></tr>`,
    )
    .join('')

  const agendaHtml = agenda
    .map((a, i) => {
      const d = decisionByAgenda[a.id]
      return (
        `<tr><td>${i + 1}</td><td><strong>${a.title}</strong>` +
        (a.description ? `<br><small>${a.description}</small>` : '') +
        `</td><td>${d?.decision_text ?? '—'}</td></tr>`
      )
    })
    .join('')

  const signedLine = chairSignedAt
    ? `Signert av ${chairName ?? '—'} den ` +
      new Date(chairSignedAt).toLocaleDateString('nb-NO', {
        day: 'numeric', month: 'long', year: 'numeric',
      })
    : ''

  return `<!DOCTYPE html>
<html lang="nb">
<head>
<meta charset="UTF-8">
<title>AMU-referat — ${title}</title>
<style>
  body { font-family: Arial, sans-serif; font-size: 12pt; margin: 2cm; color: #000; }
  h1 { font-size: 18pt; margin-bottom: 4pt; }
  h2 { font-size: 13pt; margin-top: 20pt; border-bottom: 1px solid #ccc; padding-bottom: 3pt; }
  table { border-collapse: collapse; width: 100%; margin-top: 8pt; }
  th { background: #f0f0f0; text-align: left; padding: 5pt 8pt; font-size: 10pt; }
  td { padding: 5pt 8pt; border-bottom: 1px solid #e0e0e0; vertical-align: top; }
  .meta { color: #555; font-size: 10pt; margin-bottom: 12pt; }
  .minutes { white-space: pre-wrap; border: 1px solid #ccc; padding: 10pt; margin-top: 8pt; }
  .footer { margin-top: 30pt; font-size: 10pt; color: #555; }
  @media print {
    body { margin: 1.5cm; }
    a { text-decoration: none; color: inherit; }
  }
</style>
</head>
<body>
<h1>AMU-referat: ${title}</h1>
<p class="meta">${dateLabel}${location ? ' · ' + location : ''}</p>

<h2>Deltakere</h2>
<table>
<thead><tr><th>Navn</th><th>Rolle</th><th>Til stede</th></tr></thead>
<tbody>${participantsHtml}</tbody>
</table>

<h2>Saksliste og vedtak</h2>
<table>
<thead><tr><th>#</th><th>Sak</th><th>Vedtak</th></tr></thead>
<tbody>${agendaHtml}</tbody>
</table>

<h2>Referat</h2>
<div class="minutes">${minutesDraft || '(Ingen referattekst registrert)'}</div>

<div class="footer">
  <p>${signedLine}</p>
  <p>Generert fra Klarert den ${new Date().toLocaleDateString('nb-NO', {
    day: 'numeric', month: 'long', year: 'numeric',
  })}</p>
</div>
<script>window.addEventListener('load', () => setTimeout(() => window.print(), 500))</script>
</body>
</html>`
}
```

## Anti-patterns

- **Ikke** legg til `jspdf`, `pdfmake` eller `@react-pdf/renderer` — disse er
  tunge libs som ikke er i `package.json`.
- **Ikke** bruk `document.execCommand('print')` — det er utdatert.
- **Ikke** generer PDF server-side uten eksplisitt godkjenning fra produkteier.
- **Ikke** vis eksportknappen for usignerte møter — referater kan da endres
  og PDF-en ville bli ugyldig.
- **Ikke** bruk `innerHTML` uten `DOMPurify` på bruker-generert innhold i selve
  appen. I en ny blank fane (`window.open`) er dette akseptabelt fordi
  innholdet ikke injiseres i Klarert-domenet.
