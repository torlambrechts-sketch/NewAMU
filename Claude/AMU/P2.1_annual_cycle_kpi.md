# P2.1 — Ingen årshjul eller møtefrekvensvarsel (AML §7-2 og §7-3)

## Prioritet: P2 (Medium verdi)

## Problem

`AmuPage.tsx` viser fire KPI-kort (linjene 89–104): Totalt møter, Planlagt,
Aktivt, Signert. Ingen av disse sier noe om om organisasjonen oppfyller lovkravet
om minimum fire ordinære møter per kalenderår (AML §7-3).

Det finnes ingen «fullført i år»-metrikk og ingen advarsel når årstallet
nærmer seg slutten uten at de nødvendige møtene er holdt.

Sammenlign: KaiNexus og SafetyCulture viser møteprogresjon mot et kalenderårsmål
med grønt/gult/rødt varselsymbol.

### Referansefiler (les disse FØR du redigerer noe)

- `modules/amu/AmuPage.tsx` — linjene 89–104 (`kpiItems`), 287–296 (side-shell)
- `modules/amu/useAmu.ts` — linjene 42–63 (`refresh`)
- `modules/amu/types.ts` — `AmuMeeting`, `AmuMeetingStatus`

## Acceptance criteria

- [ ] KPI-listen i `AmuPage` inneholder et nytt kort: «Fullført i år» som teller
  møter med status `completed` eller `signed` i inneværende kalenderår.
- [ ] Hvis «Fullført i år» < 4, vises en `WarningBox` under KPI-kortene med tekst
  som siterer AML §7-3 og viser hvor mange møter som gjenstår.
- [ ] Hvis «Fullført i år» >= 4, vises en `InfoBox` som bekrefter at lovkravet
  er oppfylt.
- [ ] «Inneværende kalenderår» bestemmes av `new Date().getFullYear()` klientsiden.
- [ ] Ingen databaseendring — beregningen gjøres på eksisterende
  `amu.meetings`-liste.
- [ ] `npm run build` exits 0.

## Implementeringstrinn

### Trinn 1 — Beregn «Fullført i år»

Les `AmuPage.tsx` linjene 89–104. Erstatt `kpiItems`-beregningen med:

```tsx
const kpiItems = useMemo(() => {
  let scheduled = 0
  let active = 0
  let signed = 0
  let completedThisYear = 0
  const thisYear = new Date().getFullYear()

  for (const m of amu.meetings) {
    if (m.status === 'scheduled') scheduled++
    else if (m.status === 'active') active++
    else if (m.status === 'signed') signed++

    if (
      (m.status === 'completed' || m.status === 'signed') &&
      new Date(`${m.date}T12:00:00`).getFullYear() === thisYear
    ) {
      completedThisYear++
    }
  }

  return {
    cards: [
      { big: String(amu.meetings.length), title: 'Totalt møter', sub: 'Alle AMU-møter' },
      { big: String(scheduled), title: 'Planlagt', sub: 'Ikke startet' },
      { big: String(active), title: 'Aktivt', sub: 'Saksbehandling pågår' },
      { big: String(signed), title: 'Signert', sub: 'Referat låst' },
      {
        big: String(completedThisYear),
        title: `Fullført ${thisYear}`,
        sub: completedThisYear >= 4
          ? '✓ Lovkrav oppfylt (AML §7-3)'
          : `${4 - completedThisYear} gjenstår for å oppfylle AML §7-3`,
      },
    ],
    completedThisYear,
  }
}, [amu.meetings])
```

### Trinn 2 — Oppdater visning av KPI-kort

Finn stedet der `kpiItems` sendes til `ModuleRecordsTableShell` (linjene
150–160 i `AmuPage.tsx`). Endre til:

```tsx
<ModuleRecordsTableShell
  kpiItems={kpiItems.cards}
  ...
>
```

### Trinn 3 — Frekvensadvarsel under KPI-kortene

Legg til `WarningBox`/`InfoBox` rett under KPI-tabellen og over søkebaren.
I `body`-konstanten (linjene 147–280), etter `amu.error`-visningen
(linje 149) og FØR `<ModuleRecordsTableShell>`:

```tsx
{!amu.loading && (
  kpiItems.completedThisYear < 4 ? (
    <WarningBox>
      <strong>AML §7-3 — møtefrekvens:</strong> AMU skal holde ordinære møter
      minst fire ganger per år. I {new Date().getFullYear()} er{' '}
      {kpiItems.completedThisYear} møte(r) fullført.{' '}
      {4 - kpiItems.completedThisYear} møte(r) gjenstår for å oppfylle lovkravet.
    </WarningBox>
  ) : (
    <InfoBox>
      AML §7-3 møtefrekvens for {new Date().getFullYear()} er oppfylt ({kpiItems.completedThisYear} møter fullført).
    </InfoBox>
  )
)}
```

Importer `InfoBox` fra `../../src/components/ui/AlertBox` (allerede importert
via `WarningBox` på linje 17 — legg til `InfoBox` i samme import).

### Trinn 4 — Bygg

Kjør `npm run build`. Rett alle TypeScript-feil.

## Anti-patterns

- **Ikke** tell `scheduled` eller `active` møter som "fullført" — de er ikke
  gjennomført.
- **Ikke** hent møter med et nytt Supabase-kall — bruk eksisterende
  `amu.meetings`-liste.
- **Ikke** vis advarselen i detaljvisningen — kun i listeoversikten.
- **Ikke** hard-kode årstall — bruk alltid `new Date().getFullYear()`.
