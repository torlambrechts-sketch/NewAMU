# P2.2 — Kildetittel vises ikke i sakslisten (kun modulnavn)

## Prioritet: P2 (Medium verdi)

## Problem

Agendapunkter kan knyttes til en kildemodul via `source_module` og `source_id`.
I `AmuAgendaPlanningTable.tsx` (linje 114) kaller tabellen `sourceLabel(a)`, som
kun returnerer modulnavnet ("Avvik", "Sykefravær", "Varsling") — aldri tittelen
på den faktiske saken.

Brukeropplevelsen er: sekretæren ser "Avvik" i kildekolonnen, men vet ikke
*hvilket* avvik det refereres til. I møterommet er dette meningsløst.

Tilsvarende problem finnes i `AmuMeetingRoomTab.tsx`.

### Referansefiler (les disse FØR du redigerer noe)

- `modules/amu/AmuAgendaPlanningTable.tsx` — linje 114 (`sourceLabel(a)`)
- `modules/amu/AmuMeetingRoomTab.tsx` — linjene 74–76 (sakstittel-visning)
- `modules/amu/AmuDetailView.tsx` — linjene 204–208 (`sourceLabelForAgenda`),
  617–627 (props til `AmuAgendaPlanningTable`)
- `modules/amu/useAmu.ts` — linjene 461–497 (`insertAgendaItem`) for å forstå
  `source_id`-feltets semantikk
- `modules/amu/types.ts` — `AmuAgendaItem`

## Acceptance criteria

- [ ] Ny funksjon `fetchSourceTitle(sourceModule, sourceId)` i `useAmu.ts`
  henter tittelen til den koblede posten fra riktig tabell.
- [ ] For `source_module = 'avvik'`: hent `title` fra `avvik`-tabellen.
- [ ] For `source_module = 'sick_leave'` eller `'whistleblowing'`: returner
  `null` (ingen individuell kobling av personvernhensyn).
- [ ] `AmuDetailView.tsx` henter titler for alle agendapunkter med `source_id`
  etter innlasting og lagrer dem i et `sourceTitleMap`-state.
- [ ] `sourceLabelForAgenda` bruker `sourceTitleMap` for å vise faktisk tittel
  når den finnes, ellers modulnavn som fallback.
- [ ] Hvis `source_id` er satt men henting feiler (sak slettet), vises
  `"[Slettet avvik]"` som tittel.
- [ ] `npm run build` exits 0.

## Implementeringstrinn

### Trinn 1 — Ny funksjon `fetchSourceTitle` i `useAmu.ts`

Legg til etter `fetchSickLeaveAgendaStats` (ca. linje 202):

```typescript
const fetchSourceTitle = useCallback(
  async (sourceModule: string, sourceId: string): Promise<string | null> => {
    if (!supabase || !orgId) return null
    if (sourceModule === 'sick_leave' || sourceModule === 'whistleblowing') {
      return null  // Personvern — ingen individuell kobling
    }
    if (sourceModule === 'avvik') {
      try {
        const { data, error: qErr } = await supabase
          .from('avvik')
          .select('title')
          .eq('organization_id', orgId)
          .eq('id', sourceId)
          .maybeSingle()
        if (qErr) throw qErr
        if (!data) return '[Slettet avvik]'
        return (data as { title: string }).title ?? '[Uten tittel]'
      } catch {
        return '[Slettet avvik]'
      }
    }
    return null
  },
  [supabase, orgId],
)
```

Eksporter `fetchSourceTitle` i `useMemo`-returnen og deps-arrayen (linjene
866–928).

### Trinn 3 — Hent titler etter agendainnlasting

I `refreshMeetingBundle`-callbacken (linjene 134–164), etter at `setAgenda(items)`
er kalt, legg til:

```tsx
// Hent kildetitler for alle punkter med source_module + source_id
const titlesToFetch = items.filter(
  (it) => it.source_module && it.source_id,
)
if (titlesToFetch.length > 0) {
  const entries = await Promise.all(
    titlesToFetch.map(async (it) => {
      const title = await amu.fetchSourceTitle(
        it.source_module!,
        it.source_id!,
      )
      return [it.id, title] as [string, string | null]
    }),
  )
  const map: Record<string, string> = {}
  for (const [id, title] of entries) {
    if (title !== null) map[id] = title
  }
  setSourceTitleMap(map)
}
```

## Anti-patterns

- **Ikke** hent individuelle varslingssaker eller sykefraværsposter direkte —
  personvern.
- **Ikke** hent titler én etter én i en sekvensell løkke — bruk `Promise.all`.
- **Ikke** kast feil dersom avvik-tabellen ikke eksisterer — fang og returner
  `null`.
- **Ikke** cache titler mellom ulike møter — `sourceTitleMap` nullstilles ved
  hvert `refreshMeetingBundle`-kall.
