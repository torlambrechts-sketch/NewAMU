# P0.1 — Kilde-ID-feltet er ikke-funksjonelt (råt UUID-input)

## Prioritet: P0 (Brutt UX)

## Problem

I agendapanelet i `AmuDetailView.tsx` (linjene 808–819) og i
`AmuModuleAdminPage.tsx` (linjene 264–278) er det et fritekstfelt der brukeren
skal skrive inn en UUID for å koble en sak til en konkret post i en annen modul
(avvik, sykefravær, varsling). Ingen bruker vet UUIDs til konkrete poster
utenat. Feltet er i praksis ubrukelig og gir falsk trygghet om en kobling som
aldri settes.

I tillegg vises kildemodulfeltet som en tom `—` i saklisten selv om
`source_module` er satt, fordi ingen kode henter faktisk tittel fra kildens
tabell.

### Trinn 2 — Ny funksjon i `modules/amu/useAmu.ts`

Legg til etter `fetchSickLeaveAgendaStats` (ca. linje 202):

```typescript
const loadAvvikOptions = useCallback(async (): Promise<AvvikOption[]> => {
  if (!supabase || !orgId) return []
  try {
    const { data, error: qErr } = await supabase
      .from('avvik')
      .select('id, title')
      .eq('organization_id', orgId)
      .order('created_at', { ascending: false })
      .limit(200)
    if (qErr) throw qErr
    return (data ?? []).map((r) => ({
      id: r.id as string,
      title: (r.title as string) ?? '(uten tittel)',
    }))
  } catch (err) {
    // avvik-modulen er valgfri — ikke sett global feil
    return []
  }
}, [supabase, orgId])
```

Eksporter `loadAvvikOptions` i `useMemo`-returnen og deps-arrayen på linjene
866–928.

Definer `AvvikSourcePicker` som en ny lokal komponent i `AmuDetailView.tsx`
(over eksport-funksjonen):

```tsx
function AvvikSourcePicker({
  value,
  onChange,
  loadOptions,
}: {
  value: string
  onChange: (id: string) => void
  loadOptions: () => Promise<AvvikOption[]>
}) {
  const [opts, setOpts] = useState<{ value: string; label: string }[]>([])
  useEffect(() => {
    void (async () => {
      const raw = await loadOptions()
      setOpts([
        { value: '', label: 'Velg avvik (valgfri)' },
        ...raw.map((r) => ({ value: r.id, label: r.title })),
      ])
    })()
  }, [loadOptions])

  return (
    <div className={WPSTD_FORM_ROW_GRID}>
      <span className={WPSTD_FORM_FIELD_LABEL}>Koble til avvik</span>
      <SearchableSelect value={value} options={opts} onChange={onChange} />
    </div>
  )
}
```

Importer `AvvikOption` fra `./types` og legg til `useState` (allerede
importert).

## Anti-patterns

- **Ikke** behold UUID-fritekstfeltet som "fallback" — det gir falsk trygghet
  og brukes ikke.
- **Ikke** hent avvik for `sick_leave`/`whistleblowing` — personvern.
- **Ikke** bruk `as any` for avvik-data fra Supabase — cast eksplisitt.
- **Ikke** gjør `loadAvvikOptions` obligatorisk i `AmuAgendaPlanningTable` —
  tabellen har ikke kontekst til å hente data selv.
