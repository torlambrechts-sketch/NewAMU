# P1.1 — Referat mister innhold ved navigering (ingen auto-lagring)

## Prioritet: P1 (Høy daglig påvirkning)

## Problem

Referatfanen i `AmuDetailView.tsx` (linjene 640–675) er et enkelt `StandardTextarea`
med en manuell "Lagre referat"-knapp. Hvis sekretæren skriver et langt referat
og navigerer til en annen fane — eller lukker nettleserfanen — er alt innhold
tapt uten varsel. Det finnes ingen indikator på ulagret innhold, ingen
auto-lagrings-mekanisme og ingen "er du sikker?"-dialog ved navigering.

Sammenlign: Notion, Confluence, og SafetyCulture auto-lagrer hvert 3–5 sekund
og viser "Alle endringer lagret" eller "Lagrer…" i sanntid.

### Referansefiler (les disse FØR du redigerer noe)

- `modules/amu/AmuDetailView.tsx` — linjene 127 (`minutesDraft` state),
  239–248 (`onSaveMinutes`), 640–675 (referat-tab render)
- `modules/amu/useAmu.ts` — linjene 618–673 (`updateMeeting`)
- `src/components/ui/AlertBox.tsx` — `InfoBox`-komponent (for
  "lagret"-indikator)

## Acceptance criteria

- [ ] Endringer i referatteksten lagres automatisk 2 sekunder etter siste
  tastetrykk (debounce på 2 000 ms).
- [ ] En "Lagrer…"-indikator vises under tekstfeltet mens auto-lagring pågår.
- [ ] En "Alle endringer lagret"-bekreftelse vises i 3 sekunder etter vellykket
  lagring.
- [ ] Hvis brukeren navigerer bort fra referatfanen med ulagret innhold, vises
  en `window.onbeforeunload`-advarsel (nettleserens standard dialog).
- [ ] Den manuelle "Lagre referat"-knappen beholdes for eksplisitt lagring.
- [ ] Auto-lagring er deaktivert for signerte møter (`readOnly = true`).
- [ ] `npm run build` exits 0.

## Filer som skal endres

| Fil | Endring |
|-----|---------|
| `modules/amu/AmuDetailView.tsx` | Auto-lagring med debounce, visuell indikator, `beforeunload`-hook |

Ingen databaseendringer, ingen nye npm-pakker (debounce implementeres med
`useEffect` + `setTimeout`).

## Implementeringstrinn

### Trinn 1 — Ny state for auto-lagrings-status

I `AmuDetailView.tsx`, legg til to nye state-variabler etter eksisterende state
(etter linje 128):

```tsx
const [autoSaveStatus, setAutoSaveStatus] = useState<
  'idle' | 'pending' | 'saving' | 'saved' | 'error'
>('idle')
const [prevMinutesDraft, setPrevMinutesDraft] = useState('')
```

### Trinn 2 — Debounce-effekt for auto-lagring

Legg til denne `useEffect` etter `onSaveMinutes`-callbacken (etter linje 248):

```tsx
useEffect(() => {
  // Ikke auto-lagre for signerte møter eller hvis innhold ikke har endret seg
  if (readOnly || !amu.canManage || !meeting) return
  if (minutesDraft === prevMinutesDraft) return

  setAutoSaveStatus('pending')
  const t = window.setTimeout(async () => {
    setAutoSaveStatus('saving')
    const next = await amu.updateMeeting(meeting.id, {
      minutes_draft: minutesDraft || null,
    })
    if (next) {
      setMeeting(next)
      setPrevMinutesDraft(minutesDraft)
      setAutoSaveStatus('saved')
      // Nullstill til 'idle' etter 3 sekunder
      window.setTimeout(() => setAutoSaveStatus('idle'), 3000)
    } else {
      setAutoSaveStatus('error')
    }
  }, 2000)

  return () => {
    window.clearTimeout(t)
  }
  // prevMinutesDraft ekskludert bevisst: vi vil bare reagere på endringer i minutesDraft
  // eslint-disable-next-line react-hooks/exhaustive-deps
}, [minutesDraft, readOnly, amu.canManage, meeting?.id])
```

### Trinn 3 — Synkroniser `prevMinutesDraft` ved innlasting

I `refreshMeetingBundle` (linje 134–164), etter `setMinutesDraft(m.minutes_draft ?? '')`,
legg til:

```tsx
setPrevMinutesDraft(m.minutes_draft ?? '')
```

### Trinn 4 — `beforeunload`-advarsel ved ulagret innhold

Legg til denne `useEffect` i komponenten:

```tsx
useEffect(() => {
  const isDirty = minutesDraft !== prevMinutesDraft && !readOnly
  if (!isDirty) return
  const handler = (e: BeforeUnloadEvent) => {
    e.preventDefault()
    e.returnValue = ''
  }
  window.addEventListener('beforeunload', handler)
  return () => window.removeEventListener('beforeunload', handler)
}, [minutesDraft, prevMinutesDraft, readOnly])
```

### Trinn 5 — Visuell indikator i referatfanen

Erstatt linjene 662–674 (knapp-seksjonen) med:

```tsx
<div className="border-t border-neutral-200 px-4 py-3 md:px-5 flex flex-wrap items-center justify-between gap-2">
  <span className="text-xs text-neutral-500">
    {autoSaveStatus === 'pending' && 'Venter på å lagre…'}
```

### Trinn 6 — Oppdater `onSaveMinutes` for å synkronisere `prevMinutesDraft`

Erstatt `onSaveMinutes` (linjene 239–248) med:

```tsx
const onSaveMinutes = useCallback(async () => {
  if (!meeting) return
  setSaving(true)
  const next = await amu.updateMeeting(meeting.id, {
    minutes_draft: minutesDraft || null,
  })
  if (next) {
    setMeeting(next)
    setPrevMinutesDraft(minutesDraft)
    setAutoSaveStatus('saved')
    window.setTimeout(() => setAutoSaveStatus('idle'), 3000)
  }
  setLoadTick((x) => x + 1)
  setSaving(false)
}, [amu, meeting, minutesDraft])
```

## Anti-patterns

- **Ikke** bruk `lodash.debounce` — implementer med `setTimeout`/`clearTimeout`
  i `useEffect`. Ingen nye npm-pakker.
- **Ikke** auto-lagre ved hvert tastetrykk — bruk alltid 2 000 ms debounce for
  å unngå overbelastning av Supabase.
- **Ikke** fjern den manuelle lagreknappen — brukere forventer eksplisitt
  kontroll i tillegg til auto-lagring.
- **Ikke** sett `autoSaveStatus` til `'error'` uten at brukeren informeres
  (WarningBox-banneret øverst i skjemaet viser `amu.error`, men
  auto-lagringsfeilen vises separat i indikatoren).
